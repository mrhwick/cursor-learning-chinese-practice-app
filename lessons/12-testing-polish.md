## Lesson 12: Testing + polish

### Goal
Learn how to make changes safely:
- write a failing test
- make it pass
- refactor without breaking behavior

### Allowed files
- `apps/api/tests/*`
- `apps/api/chinese_practice/main.py`

### Definition of Done
- You added at least one new API test.
- You can explain what the test is protecting against.

---

## Suggested tasks

1) Add an API test that verifies unauthenticated calls to `/api/me` return 401.
2) Add an API test that verifies a student cannot create a classroom.
3) Ask Cursor to explain why tests are especially valuable with AI-assisted coding.
